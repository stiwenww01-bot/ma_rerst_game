# 📱 Промпт для разработки Android-игры "Block Puzzle Saga"

## 🎯 Краткое описание проекта

Разработать Android-приложение — головоломку в стиле Woodoku/Block Blast с механикой размещения фигур на сетке 9×9. Игрок собирает горизонтальные/вертикальные линии и квадраты 3×3 (всего 9 клеток) для получения очков. Уникальные фишки: возможность поворота фигуры (3 раза за игру) и схлопывание трёх одинаковых фигур подряд. Игра имеет 4 визуальных стиля на выбор: Космос, Вестерн, Классика (дерево), Киберпанк.

---

## 🎮 Игровая механика (детально)

### Поле
- **Размер:** 9×9 клеток (81 клетка), разделено на 9 квадратов 3×3 (как в судоку)
- **Состояние клетки:** пустая / заполненная (с цветом/текстурой по теме)

### Фигуры (тетромино + расширения)
- Базовый набор: L, J, T, S, Z, I, O, плюс одиночные блоки, линии 2/3/4/5 клеток, квадраты 2×2 и 3×3, угловые формы (всего ~25–30 типов).
- За один ход игроку предлагается **3 фигуры** в нижней панели. Когда все три размещены — генерируются новые три.
- Фигуры можно перетаскивать (drag & drop) на любое валидное место поля.

### 🔄 Поворот фигуры (фишка №1)
- **Лимит: 3 поворота за игровую сессию** (отдельная кнопка с счётчиком, например "↻ 3").
- Поворот применяется к выбранной фигуре на 90° по часовой стрелке.
- После исчерпания лимита — кнопка неактивна, можно докупить за просмотр рекламы (опционально для будущих версий).

### 🔥 Схлопывание фигур (фишка №2)
- Если игрок размещает **3 одинаковые фигуры подряд** (без других между ними), они автоматически "схлопываются" — исчезают с поля, дают бонусные очки и заполняют все клетки, которые занимали.
- Счётчик "цепочки" сбрасывается, если поставлена другая фигура.

### Очистка линий и комбо
- **Горизонтальная линия (9 клеток):** +100 очков
- **Вертикальная линия (9 клеток):** +100 очков
- **Квадрат 3×3 (9 клеток):** +100 очков
- **Комбо за один ход:**
  - 2 очистки одновременно: +50 бонус (×1.5)
  - 3 очистки: +150 бонус (×2)
  - 4 и более: +400 бонус (×3)
- **Очки за размещение фигуры:** по 1 очку за каждую клетку фигуры.

### Конец игры
- Когда ни одна из трёх доступных фигур не помещается на поле — Game Over.
- Сохраняется рекорд (общий, за месяц, за неделю, за день).

## 🎨 Визуальные стили (4 темы)

Игрок выбирает стиль в главном меню. Стиль меняет: фон, текстуру блоков, цвета UI, звуковые эффекты, шрифты.

### 1. 🌌 Космос
- Фон: тёмное звёздное небо с лёгкой анимацией мерцания звёзд и медленно дрейфующей туманностью.
- Блоки: неоновые кристаллы (фиолетовый, голубой, бирюзовый) со свечением.
- При очистке линии: эффект "взрыва сверхновой" с частицами.
- Шрифт: футуристичный (например, Orbitron).
- Звуки: космические синтетические сигналы.

### 2. 🤠 Вестерн
- Фон: пустыня, кактусы, закат, деревянная вывеска салуна.
- Блоки: деревянные ящики с клеймом, бочки, мешки золота.
- При очистке: пыль и анимация перекати-поля.
- Шрифт: стилизованный под старые афиши (например, Rye).
- Звуки: губная гармошка, выстрелы, скрип двери.

### 3. 🪵 Классика (дерево)
- Фон: текстура светлого дерева (как Woodoku).
- Блоки: тёплые деревянные плашки с волокнистой текстурой.
- При очистке: щепки и звук падающих деревяшек.
- Шрифт: округлый, дружелюбный (например, Fredoka).
- Звуки: тёплый стук дерева.

### 4. 🌃 Киберпанк
- Фон: ночной мегаполис с неоновыми вывесками, лёгкий дождь.
- Блоки: голограммы с глитч-эффектами, неоновая обводка (розовый, лазурный).
- При очистке: эффект "цифрового распада" пикселями.
- Шрифт: моноширинный, технологичный (например, Share Tech Mono).
- Звуки: синтвейв-эффекты, электронные сигналы.

## 🛠 Технический стек (с упором на простоту)

### Платформа и язык
- **Android (нативно)** на **Kotlin**.
- Минимальная версия: **Android 8.0 (API 26)**.
- Целевая: **Android 14 (API 34)**.

### UI-фреймворк
- **Jetpack Compose** — современный декларативный UI, проще XML-разметки. Достаточно для всех экранов кроме игрового поля.
- **Canvas (Jetpack Compose Canvas)** для отрисовки игрового поля и фигур. Не нужен сложный игровой движок типа Unity/LibGDX — Canvas справится.

### Локальная база данных
- **Room (SQLite поверх Android)** — самая простая ORM для Android.
- Таблицы:
  - `game_state` — текущее состояние партии (поле, фигуры, очки, оставшиеся повороты)
  - `records` — рекорды (общий, дневной, недельный, месячный, годовой)
  - `settings` — выбранный стиль, звук вкл/выкл, вибрация
  - `statistics` — кол-во сыгранных партий, общее время, лучшая комбо-цепочка

### Анимации
- Стандартные анимации Compose (`animate*AsState`, `AnimatedVisibility`).
- Для эффектов частиц — простая своя реализация на Canvas (массив объектов с координатами, скоростями, прозрачностью).

### Звук
- **MediaPlayer / SoundPool** из Android SDK — простой и встроенный, без сторонних библиотек.

### Архитектура
- **MVVM** + StateFlow для реактивного UI.
- **Без DI-фреймворков** (Hilt/Koin) на старте — простая ручная инициализация. Можно добавить Hilt позже.

### Структура проекта
```
app/
├── data/
│   ├── db/              # Room: Entities, DAOs, Database
│   ├── repository/      # GameRepository, SettingsRepository
├── domain/
│   ├── model/           # Piece, Board, GameState
│   ├── logic/           # PieceGenerator, LineChecker, ScoreCalculator, ComboTracker
├── ui/
│   ├── screens/
│   │   ├── menu/        # MainMenuScreen
│   │   ├── game/        # GameScreen, GameViewModel
│   │   ├── settings/    # SettingsScreen
│   │   ├── records/     # RecordsScreen
│   ├── theme/           # 4 темы: Space, Western, Classic, Cyberpunk
│   ├── components/      # BoardCanvas, PieceTray, ScorePanel
├── audio/               # SoundManager
└── MainActivity.kt
```

## 🤖 Какие AI-агенты задействовать

### 1. **Architect Agent** (планирование архитектуры)
- Создаёт структуру проекта, описывает классы и их связи.
- Утверждает контракты между слоями (data ↔ domain ↔ ui).

### 2. **Game Logic Agent** (игровая логика)
- Реализует чистые Kotlin-классы без зависимостей от Android: `Board`, `Piece`, `LineChecker`, `ScoreCalculator`, `ComboTracker`, `PieceGenerator`.
- Покрывает их **JUnit-тестами** (это критично — логика должна быть отлажена ДО UI).

### 3. **UI/Compose Agent** (Jetpack Compose)
- Реализует экраны на Compose, навигацию (Navigation Compose), ViewModel-ы.
- Делает Canvas-отрисовку игрового поля и фигур.

### 4. **Database Agent** (Room)
- Описывает Entity, DAO, миграции. Реализует репозитории.

### 5. **Graphics/Design Agent** (Figma MCP)
- Использует **Figma MCP** для генерации макетов экранов в 4 стилях.
- Экспортирует ассеты: фоны, текстуры блоков, иконки, шрифты.
- Создаёт **дизайн-токены** (цвета, размеры, тени) для каждой темы → они переносятся в Compose Theme.

### 6. **Audio Agent**
- Подбирает / генерирует короткие SFX и фоновую музыку для каждой темы.
- Настраивает SoundManager.

### 7. **QA / Test Agent**
- Пишет unit-тесты на игровую логику.
- Пишет UI-тесты (Compose Testing) на ключевые сценарии: размещение фигуры, очистка линии, поворот, схлопывание.
- Проверяет каждый этап перед переходом к следующему.

### 8. **Release Agent** (на финальном этапе)
- Настраивает подпись APK, ProGuard/R8, иконку приложения, метаданные для Google Play.

### **ЭТАП 1. Доменная модель и игровая логика (без UI)**
**Агент:** Game Logic Agent + QA Agent.

**Цель:** реализовать всю игровую механику в виде чистых Kotlin-классов с тестами.

**Задачи:**
1. `Piece` — класс фигуры (форма как массив координат, тип, цвет).
2. `Board` — игровое поле 9×9, методы `canPlace(piece, x, y)`, `place(piece, x, y)`, `clearLines()`.
3. `LineChecker` — проверка заполненных строк / столбцов / квадратов 3×3.
4. `ScoreCalculator` — расчёт очков за размещение и за комбо.
5. `ComboTracker` — отслеживание цепочки из 3 одинаковых фигур.
6. `PieceGenerator` — генерация набора из 3 случайных фигур (с балансом по сложности).
7. `RotationManager` — учёт оставшихся поворотов (стартово 3), метод `rotate(piece)`.
8. `GameState` — агрегатор всего состояния, метод `isGameOver()`.

**Тесты (JUnit):**
- Размещение фигуры в валидных/невалидных позициях.
- Очистка одной линии, нескольких линий, квадрата 3×3.
- Комбо: 2/3/4 очистки за ход — корректные множители.
- Цепочка из 3 одинаковых фигур → схлопывание.
- Поворот фигуры → корректное преобразование координат, уменьшение счётчика.
- Game Over: ни одна фигура не помещается.

**✅ Проверка:**
- [ ] Все юнит-тесты проходят (минимум 25–30 тестов).
- [ ] Покрытие игровой логики тестами ≥ 80%.
- [ ] Логика работает независимо от Android (можно запустить в JVM).

## 📐 Правила работы для AI-агентов

1. **Не переходить к следующему этапу, пока не выполнен чек-лист текущего.**
2. **Код пишется максимально просто:** функция > класс, композиция > наследование.
3. **Никакого преждевременного DI, реактивщины ради реактивщины, абстракций "на будущее".**
4. **Каждый PR/коммит сопровождается тестом** (где применимо).
5. **Игровая логика отделена от Android SDK** — должна тестироваться в чистом JVM.
6. **Все строки выносятся в `strings.xml`** с самого начала (для локализации).
7. **Размеры и отступы — через dp, тексты — через sp.**
8. **Цвета и стили — только через Theme**, никаких хардкодов в Compose-компонентах.

## 🎯 Критерий успеха MVP

Игрок может:
- Открыть приложение, выбрать стиль из 4.
- Сыграть полную партию с механиками поворота (3 раза) и схлопывания.
- Получить очки за комбо.
- Увидеть свой рекорд в таблице.
- Закрыть приложение и продолжить партию с того же места.

Игра при этом:
- Стабильна (нет крашей за 100 партий тестирования).
- Плавна (60 FPS на устройствах среднего класса).
- Весит не более 50 МБ.
